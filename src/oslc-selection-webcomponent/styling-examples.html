Framework Integration

The component can be easily styled within any CSS framework:

Tailwind CSS Integration

/* In your CSS file */ .oslc-btn-primary { --oslc-button-background: theme('colors.blue.500'); --oslc-button-color: theme('colors.white'); --oslc-button-hover-background: theme('colors.blue.600'); --oslc-button-border-radius: theme('borderRadius.md'); --oslc-button-padding: theme('spacing.2') theme('spacing.4'); } /* In your HTML */ <oslc-selection-button class="oslc-btn-primary">

CSS-in-JS (Styled Components)

const StyledOslcButton = styled('oslc-selection-button')` --oslc-button-background: ${props => props.theme.primary}; --oslc-button-color: white; --oslc-button-border-radius: 4px; --oslc-button-hover-background: ${props => props.theme.primaryDark}; &.large { --oslc-button-padding: 16px 32px; --oslc-button-font-size: 1.2rem; } `;

Advantages of Raw Web Components vs Lit

✅ Raw Web Components (Current)

  • Zero dependencies - No framework overhead
  • Smaller bundle size - ~5KB vs ~25KB with Lit
  • Full CSS control - Complete customization via CSS variables
  • Framework agnostic - Works everywhere
  • Predictable styling - No framework-specific styling quirks
  • Better for JSP - No build step required

⚠️ Lit Framework Alternative

  • Framework dependency - Adds ~20KB to bundle
  • More complex build - Requires additional tooling
  • CSS abstractions - Less direct control over styling
  • Learning curve - Framework-specific patterns
  • Overkill for simple component - Our use case is straightforward

Conclusion: For OSLC Selection component, raw Web Components provide better developer experience, smaller footprint, and more flexible styling options. The comprehensive CSS custom properties system gives users complete control over appearance without framework dependencies.
